
## 커밋 컨벤션

```
커밋 유형: 제목
---
- 내용1
  설명
- 내용2
  설명

(이슈번호)
```


<br>

### 커밋 유형 종류

| 커밋 유형 | 의미 |
| --- | --- |
| `feat`  | 새로운 기능 추가 또는 기능 개선 |
| `refactor` | 중복 메소드 제거, 클래스 책임 분리, 메소드 책임 분리 등 |
| `chore`  | 단순 오타 수정, 클래스명/변수명 변경 등 간단한 작업 |
| `fix` | 버그 수정 |
| `hotfix` | 매우 중요한 버그 수정 |
| `config` | config 파일, dependency, application.yml 등 설정 파일 수정 |
| `test` | 테스트 관련 작업 |
| `init`  | 프로젝트 생성 |
| `docs`  | 주석 작업, README, Swagger, StoryBook |

*해당 커밋 유형 키워드는 백엔드 개발 레포에서 적용


<br>

### 제목 규칙

- 최대한 짧게, 50자 이하
- 단어 대문자 시작
- 영문 표기 시 동사(원형) 시작
- 한글 표기 시 '-다' 가 아닌 '-ㅁ' 및 명사 표현
